Gaussian Elimination

Goal: Solve Ax=b for x

  • Solving via A−1 is slow.
  • Solving via A−1 leads to large numerical error.

Diagonal Matrices:

  • xk​=dk​bk​​, where dk​ is the kth element of the matrix's diagonal.

Upper Triangular Matrices:

  • Use the familiar backward substitution.

Lower Triangular Matrices:

  • Use forward substitution.

Gaussian Elimination:

  • Transform a matrix into row-reduced echelon form.
  • Apply backwards substitution.
  • Pivot Element - Entry on the diagonal.
    • Leading entry per row.
    • Used to zero out lower entries.
  • Multiplier - Used to zero out lower entries.
    • mult=pivot elemententryunderpivot​
    • One multiplier per row (for the leading element only) under the pivot element.
    • Used in the "subtract a multiple of one row from another" elementary operation.
      • Multiple the pivot element's row by the multiplier and subtract that from another (below) row.
  • Multipliers and row subtraction can be encoded in a lower triangular matrix.

LU Decomposition

Gaussian elimination is equivalent to finding matrices L & U such that:

  • L is a unit lower triangular matrix that encodes the multipliers and row subtraction operations mentioned above.
  • U is the upper triangular matrix that results from applying the above operations to A.
  • A=LU.
  • The LU decomposition exists and is unique iff all the leading principal submatrices of A are nonsingular.
    • i.e. It is not always possible to find an LU decomposition for an invertible matrix.
  • Leading Principal submatrices are the k×k submatrices of an n×n matrix for k=[1,n].

Pivoting - LUP Decomposition

  • If A is invertible, it is always possible to find a permutation P such that PA=LU.
    • i.e. An LU decomposition exists for PA iff A is invertible.
  • Permutation Matrix - Encodes swapping of rows of A in a matrix.
  • P−1=PT.
  • PA permutes rows.
  • APT permutes columns. PT is somtimes written as Q.
  • The permutation matrix can be stored as a single vector.
    • Entries are the index (starting at 1) of the "1" entry in each row of the permutation matrix.
  • Not every matrix has an LU decomposition: A=(01​11​)
  • The leading 0 in the first row prevents simple Gaussian Elimination or LU Decomposition.
  • The solution is to "pivot" (swap rows).
    • Encode this in a permutation matrix.
  • Pivot rows so that the largest element in each column is at the pivot position.
  • Swap the rows in:
    • The pivot matrix - to keep track of row swaps.
    • The (in-progress) U matrix.
    • The (in-progress) L matrix.
      • Swap the elements in the corresponding rows up to, but excluding the, the pivot column (i.e. the entry on the diagonal).
  • To solve Ax=b from PA=LU:
    • Multiply b by P.
    • Solve Lx1​=Pb using forward substitution.
    • Solve Ux2​=x1​ using backward substitution.
    • x2​ is the solution to Ax=b.
  • Pivoting keeps the error small.
    • Stabilises differences in elimination.
  • Pivoting keeps the multipliers in the range (0,1].
  • Partial Pivoting - Swapping rows to use small multipliers.
    • "Complete" pivoting would also swap columns.

Errors and Residuals

The Error is defined as: e=x−x∗​Error vector∣∣e∣∣Error

The Relative Error is defined as: ∣∣x∣∣∣∣x−x∗​∣∣​x=0

The Residual is defined as: r=b−Ax∗​Residual vector∣∣r∣∣Residual

The Relative Residual is defined as: ∣∣b∣∣∣∣b−Ax∗​∣∣​b=0

  • Norm-wise errors can hide component-wise errors in vectors.
    • One component may have significant error, but the overall relative error may not be as large.
  • Generally, x is unknown so e is not computable.
  • Condition Number (K) - A measure of the relative error with respect to the relative residual.
    • Can derive an upper bound for relative error. Relative Error≤K⋅Relative Residual
  • The relative error is at most K⋅Relative Residual.
  • Rule of thumb: The order of magnitude of K tells you how many fewer digits for x∗​ you can compute than you have for b.
